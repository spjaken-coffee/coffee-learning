# Coffee Learning App MVP Roadmap

## 1. 目的

コーヒーの学習を「読む」だけでなく、「選ぶ」「考える」「結果を知る」体験に変える。
特に精製方法、焙煎、挽き目、抽出の因果関係を、分岐型の学習フローで楽しく記憶に残る形にする。

## 2. 想定する MVP

最初の公開版では、以下を完成ラインとする。

1. 学習テーマを自分で選べるホーム画面
2. 学習マップ画面
3. コーヒーチェリー構造の図解コンテンツ
4. 精製方法を分岐選択で学べるシミュレーター
5. 生豆から焙煎までの基礎チャプター
6. 挽き目と抽出の関係を学べる基礎チャプター
7. 各チャプターから次に学ぶべき導線

## 3. 学習体験の柱

### A. 図で理解する

- コーヒーチェリーの構造
- 精製工程の流れ
- 焙煎度による変化
- 粒度と抽出速度の関係

### B. 選んで理解する

- 「果肉を残すか」
- 「水洗いするか」
- 「嫌気発酵させるか」
- 「浅煎りにするか深煎りにするか」
- 「粗挽きか細挽きか」

### C. 結果で記憶に残す

- 最後に到達した精製方法
- その方法の特徴
- 風味への影響
- 代表産地
- 次に学ぶべき関連テーマ

## 4. 画面構成案

### 1. ホーム `/`

- アプリの価値提案
- 学べるテーマ一覧
- おすすめの学習順
- 「まず始める」導線

### 2. 学習マップ `/learn`

- チャプター一覧
- 初学者向けの推奨ルート
- 各チャプターへの入口

### 3. チェリー構造 `/learn/cherry`

- 断面図
- 各層の説明
- 精製方法とのつながり

### 4. 精製シミュレーター `/learn/processing-simulator`

- 分岐型の選択
- 選択履歴の表示
- 到達した精製方法の解説
- 風味と代表産地の表示

### 5. 焙煎チャプター `/learn/roasting`

- 浅煎りから深煎りの違い
- 酸味、甘み、苦味、香りの変化
- 精製方法との組み合わせ例

### 6. 挽き目と抽出 `/learn/brewing`

- 挽き目と抽出速度
- 過抽出・未抽出の考え方
- 器具ごとの目安

## 5. 実装順

### Phase 1. 土台づくり

1. アプリ名、メタデータ、トップページ文言をコーヒー学習用に変更
2. 共通デザイン方針を整理
3. チャプターデータを共通管理できる形にする

### Phase 2. 学習導線の整備

1. ホーム画面を教材アプリ仕様に刷新
2. 学習マップを MVP 基準で整理
3. チャプター間リンクを明確化

### Phase 3. 体験コアの強化

1. 精製シミュレーターに学習的な説明を追加
2. 選択理由と結果のつながりを強化
3. 到達結果から次の学習先へ誘導

### Phase 4. 残りチャプターの実装

1. 焙煎の基礎ページ作成
2. 挽き目と抽出の基礎ページ作成
3. 総復習チャプターの雛形作成

### Phase 5. 公開準備

1. アイコンとタイトル整備
2. ローカル公開方法の確定
3. 必要なら PWA かデスクトップ化を判断

## 6. 追加したい拡張要素

- 学習進捗の保存
- クイズモード
- 焙煎と抽出の分岐シミュレーター
- 風味マップ
- 産地別の違い比較

## 7. 先に決めたいこと

以下は早めに決めると実装がぶれにくい。

1. 公開形式はブラウザアプリか、インストール型ローカルアプリか
2. 対象ユーザーは初心者中心か、バリスタ学習者まで含むか
3. 最初の公開版は日本語のみか
